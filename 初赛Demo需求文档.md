## 一、初赛核心要求（摘自官方指南）

### 1.1 初赛任务

将报名时的创意做成一个**可交互、能体验的 Demo**，到[初赛专区](https://forum.trae.cn/c/38-category/40-category/40)发 Demo 作品帖。

**参赛前提**：报名审核**必须已通过**，否则 Demo 不进入评审。

### 1.2 Demo 作品帖格式

| 要素 | 要求 |
|------|------|
| **标签** | `学习工作`（与报名通过的赛道一致） |
| **标题** | `【学习工作赛道】视频知识萃取仪 Demo — ...` |
| **正文** | 至少包含以下 4 部分（可增加额外部分） |

**正文必须包含的 4 个部分：**

1. **Demo 简介** — 产品形态、核心用户、2-3 个核心功能（配截图）
2. **Demo 创作思路** — 灵感来源、要解决的问题、方向判断
3. **Demo 体验地址** — 交互式 HTML 文件（Zip 打包上传社区）
4. **TRAE 实践过程** — 开发关键步骤截图（≥3 张）+ Session ID（≥3 个）+ 报名帖链接

### 1.3 评审维度（4 个）

| 维度 | 说明 | 权重分析 |
|------|------|----------|
| **创意价值** | 选题是否真实、是否解决实际问题、是否有社会或商业价值 | 最重要——痛点必须具体 |
| **完成度** | Demo 能否运行、核心功能是否可用、体验是否流畅 | 核心——能跑通、能体验 |
| **TRAE 使用深度** | 是否充分利用 TRAE 的能力、创作过程是否清晰可追溯 | 关键——截图+Session ID |
| **表达清晰度** | 作品帖结构是否完整、描述是否清晰、截图和 Session ID 是否充分 | 基础要求 |

### 1.4 关键约束

- 作品必须**全部使用 TRAE Work 或 TRAE IDE** 创作
- 需提供**可体验的 HTML 文件**（Zip 打包上传），打开即用，无需部署
- 需附**≥3 个 Session ID**，证明由 TRAE 完成
- 需附**≥3 张开发关键步骤截图**
- 提交截止后不得擅自补交、替换、撤回或修改

## 二、Demo 功能范围（MVP 定义）

### 2.1 核心定位

> **一句话**：用户粘贴 B 站视频链接，Demo 模拟 AI 萃取过程，展示结构化笔记的输出效果。

### 2.2 必须实现的功能（P0）

| # | 功能 | 说明 | 优先级 |
|---|------|------|--------|
| 1 | **链接输入** | 一个输入框，用户可粘贴 B 站/小红书视频链接 | P0 |
| 2 | **模拟萃取流程** | 点击按钮后，展示"提取中→分析中→生成中"的加载状态，模拟 AI 处理过程 | P0 |
| 3 | **结构化笔记展示** | 输出层级分明的笔记：标题、章节大纲、重点摘要、核心观点 | P0 |
| 4 | **可视化切换开关** | 一个开关按钮，"开启可视化"后笔记切换为知识卡片样式（卡片/导图布局） | P0 |
| 5 | **数据示例** | 内置 2-3 条示例视频数据，用户无需真实链接也能体验效果 | P0 |

### 2.3 建议实现的功能（P1）

| # | 功能 | 说明 | 优先级 |
|---|------|------|--------|
| 6 | 笔记导出 | 支持复制笔记文本 / 下载知识卡片图片 | P1 |
| 7 | 笔记历史 | 之前生成的笔记显示在侧边栏，点击可回顾 | P1 |
| 8 | 推荐模块 | 基于当前视频主题，展示 2-3 条推荐视频（用模拟数据） | P1 |
| 9 | 暗色/亮色模式切换 | 提升 Demo 的完成度印象 | P1 |

### 2.4 暂不实现的功能（留到复赛）

| 功能 | 原因 |
|------|------|
| 真实 B 站 API 对接 | 涉及平台合规，Demo 阶段用模拟数据 |
| 真实 AI 模型接入（LLM 概括） | Demo 阶段用前端模拟效果展示 |
| 用户登录/账号系统 | 超出 Demo 范围 |
| 后端服务部署 | Demo 为纯前端 HTML，零依赖 |

## 五、Demo 的技术方案

### 5.1 技术栈

| 层面 | 方案 |
|------|------|
| **语言** | 纯 HTML + CSS + JavaScript |
| **框架** | 无（纯原生，零依赖，打开即用） |
| **数据** | 内置 JSON 模拟数据 |
| **存储** | localStorage（笔记历史） |
| **打包** | 单个 HTML 文件（或 index.html + data.js 的极小结构） |

### 5.2 核心交互逻辑

```
输入链接 → 匹配预置数据（根据关键词触发不同示例）
         → 展示加载动画（setTimeout 模拟处理延迟）
         → 渲染笔记结果（文字模式 / 卡片模式）
         → 切换可视化开关（CSS class 切换布局）
```

### 5.3 可视化模式的前端实现

- **文字笔记模式**（默认）：左侧大纲 + 右侧详情的两栏布局
- **可视化卡片模式**（切换后）：以知识卡片 UI 展示核心观点，每张卡片一个主题，类似"闪卡"效果
- 切换时用 CSS transition 做平滑过渡动画

### 5.4 不需要的东西

- ❌ 不需要后端服务器
- ❌ 不需要数据库
- ❌ 不需要真实的 AI API 调用
- ❌ 不需要用户登录系统
- ❌ 不需要网络请求

## 六、TRAE 开发计划（3 轮对话）

### 第 1 轮：搭建核心框架

**目标**：完成 Demo 的基础骨架和核心交互

**具体任务**：
- 页面布局：标题区 + 输入框 + 结果展示区 + 底部
- 输入框 + "开始萃取"按钮 + "试试示例"按钮
- 加载动画（3 阶段模拟进度）
- 笔记结果展示（文字模式）
- 内置 1 条示例数据

### 第 2 轮：完善交互与数据

**目标**：补齐全部示例数据，增加可视化切换和体验打磨

**具体任务**：
- 补齐 3 条预置示例数据
- 可视化模式切换（卡片展示布局）
- 笔记导出功能（复制文本）
- 推荐模块（基于示例数据的推荐展示）

### 第 3 轮：打磨体验与截图准备

**目标**：视觉美化、动画打磨、准备评审材料

**具体任务**：
- 视觉风格统一（深色/浅色主题）
- 入场动画、切换过渡动画
- 响应式适配（手机端可用）
- 截图 3 张（关键步骤记录）
- 记录 Session ID
