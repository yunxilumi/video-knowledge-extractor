## 1. Demo 简介

**产品形态**：Web 应用（AI 视频知识萃取工具），交互式 HTML 文件，打开即用，无需部署。

**面向谁**：
- 🎓 **备考学生**：在 B 站看考研/考证/留学教程，需要高效提炼重点、生成复习笔记
- 💼 **职场学习者**：在小红书/B 站学技能（PPT、Excel、编程等），看完了但记不住
- ✍️ **内容创作者**：需要参考竞品视频结构，快速分析优质视频的内容框架
- 📚 **知识收藏爱好者**：收藏了一大堆"干货视频"但从未回顾

**核心功能**：

| 功能 | 说明 |
|------|------|
| 🔗 视频链接 → 结构化笔记 | 粘贴 B站/小红书视频链接，AI 模拟萃取过程，输出章节大纲+核心观点+重点摘录 |
| 🎨 知识卡片可视化 | 一键切换知识流时间线视图，支持导出为高清 PNG 图片（集成 html2canvas） |
| ⭐ 收藏系统 | 收藏喜欢的笔记，统一管理 |
| 📋 历史记录 | 自动保存每次萃取记录，点击可直接回顾笔记 |
| 🎯 兴趣画像 | 20 个标签自由选择，推荐系统根据你的兴趣加权排序 |
| 📅 每日精选 | 基于兴趣标签 + 浏览历史的个性化推荐 |
| 🔍 搜索功能 | 实时搜索视频标题、标签、作者 |

## 2. Demo 创作思路

**灵感来源**：
我自己就是 B 站和小红书的深度用户，收藏夹里躺着几十个"必看教程"，但大部分从未被打开第二次。我发现一个很普遍的困扰：**收藏了不等于学会了，看过了不等于记住了。** 视频是线性播放的，但知识是结构化的——二者天然不匹配。我想要一个工具，能帮我把视频里的"干货"真正提取出来，变成可以随时回顾、搜索、分享的知识笔记。

**想解决的问题**：
问题的关键不在于内容不够好，而在于**没有一种方式让视频里的知识真正留存下来**。现有的收藏功能只是"打了个标签"，没有帮用户完成"信息→知识"的转化。每次想回顾某个知识点，要在收藏夹里翻半天；每次学完一个教程，过几天就忘得差不多。

**为什么做这个方向**：
市面上有各种笔记工具和 AI 助手，但它们要么需要手动复制粘贴内容，要么输出纯文字缺乏可视化。没有一个产品专门做"视频→结构化知识"的转化链条——同时支持文字笔记和可视化卡片，而且把选择权交给用户。AI 的文本理解与概括能力在 2026 年已经足够成熟，但市场上仍然缺少这样一个聚焦"视频知识转化"的轻量工具。

## 4. TRAE 实践过程

### 开发流程概览

整个 Demo 用 **TRAE Work** 分 4 轮对话完成开发，历时两天：

**第 1 轮：创意概念 + 报名文档**
- 从零开始，通过对话逐步梳理产品定位、核心功能、目标用户
- 生成创意概念文档和报名帖文档
- 生成创意提案 HTML 页面（含 Hero 区、粒子背景、ScrollTrigger 动画）

**第 2 轮：搭建 Demo 核心框架**
- 实现链接输入 + 示例视频按钮 + 3 阶段萃取加载动画
- 内置 3 条完整模拟数据（考研政治 / PPT技巧 / ChatGPT原理）
- 结构化文字笔记展示（章节大纲 + 核心观点 + 重点摘录 + 时间戳）
- 文字/知识卡片双模式切换

**第 3 轮：丰富功能 + 视觉升级**
- 集成 tsParticles 粒子动态背景、GSAP 和 Anime.js 专业动画引擎
- 增加收藏系统、历史记录、智能推荐引擎（基于标签权重）
- 集成 html2canvas 实现知识卡片导出为高清图片
- 深色/浅色主题切换、3D 卡片悬停效果

**第 4 轮：精细化打磨**
- 左侧固定导航栏重构（替代浮动按钮）
- 知识流时间线布局（替代方框卡片网格）
- 兴趣画像系统（20 个标签自由选择）
- 每日精选杂志视图 + 搜索功能
- 图标更换为 Lucide Icons 专业矢量图标库
- 加载性能优化 + CDN 故障降级处理

## 5. 经验总结与心得

### 我的背景

我是一名**大数据技术专业的大学生**，对编程代码并不精通。说实话，让我自己从头写这样一个 Demo，几乎是不可能的事。但是在 TRAE Work 的帮助下，我只需要把自己的想法一步一步用自然语言告诉它——"我想要一个输入链接的地方"、"能不能加一个加载动画"、"这个颜色太暗了"……TRAE 就能理解并实现出来。

整个过程花了**两天时间**，从最初的创意概念，到最终的完整可交互 Demo，完全是通过对话聊天完成的。有时候我提出一个需求，TRAE 甚至会主动给出更好的建议，比如推荐集成 tsParticles 做动态背景、用 GSAP 做入场动画、用 Lucide Icons 替换 emoji 图标等等。

### 用 TRAE Work 做 Demo 的感受

1. **零门槛上手**：不需要懂 HTML、CSS、JavaScript，只要会描述自己想要的，TRAE 就能做出来
2. **迭代极其方便**：觉得哪里不满意，直接说"改这里"，几秒钟就看到效果
3. **专业能力惊人**：从粒子动画到图片导出，从推荐算法到图标库，TRAE 知道用什么技术方案最好

### 踩过的坑

1. **CDN 加载失败导致按钮不响应**：一开始把 GSAP、Anime.js 等动画库放在 `<head>` 同步加载，如果网络不好动画库加载失败，整个脚本就中断了。后来加了 `try/catch` 降级处理，即使 CDN 全部加载失败，核心功能依然可用。
2. **颜色太暗看不清**：第一版是纯黑背景（`#06080c`），整体太暗。后来改成深蓝灰色（`#0f172a`），对比度更好，视觉更舒适。
3. **功能越加越多失去焦点**：一开始想加很多功能，差点没控制住范围。后来回归 MVP 思维，先保证核心链路跑通，再逐步扩展。

### 给其他参赛者的建议

- **先想清楚核心功能再动手**：先想好你的产品解决什么问题，再让 TRAE 帮你实现
- **不会代码也能参赛**：我就是活生生的例子——只会描述需求，TRAE 就能帮你做出来
- **保留每一次对话记录**：Session ID 是评审验证作品真实性的关键，务必在创作过程中保留
- **不要追求完美**：先做出来一个能跑通的版本，再慢慢优化——TRAE 的迭代能力很强
